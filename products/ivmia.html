Key Features

Unified Dashboard

See all hosts, workloads, and metrics across every platform in a single web interface. No more switching between vCenter, Proxmox UI, and kubectl.

Dual-Transport Architecture

Every platform supports two transport methods (SSH + API) with automatic failover. If one path goes down, collection continues via the other.

Federated Multi-Site

Deploy sub-cores at remote sites that collect locally and sync to a central core. Works across VPNs and high-latency links.

Lightweight Agents

Base agent monitors any Linux or Windows host. Platform extensions add workload-specific collection. Packages under 100KB. No Java, no heavy dependencies.

Operations Management

Start, stop, restart, snapshot, and manage workloads directly from the web UI. Confirmation dialogs on destructive operations. Full audit trail.

Analysis & Trending

CPU, memory, disk, and network trends over time. Capacity heatmaps. Cross-platform comparison radar charts. All powered by Chart.js with AJAX auto-refresh.

Architecture

Designed for heterogeneous environments and distributed deployments.

Central Core (PostgreSQL + Apache/PHP + Python Collectors)
    |
    +-- Web UI (dashboards, operations, analysis, search)
    +-- REST API (FastAPI — agents, federation, RBAC)
    +-- Super Orchestrator (drives platform-specific collectors)
    |       +-- VMware ESXi, Proxmox, KVM, Hyper-V
    |       +-- Docker, Kubernetes, Incus, VirtualBox
    |
    +-- Federation Sync <-- Sub-Core (remote site)
    |                        +-- Local DB + collectors
    |                        +-- Pushes data to central
    |
    +-- Agent Manager <-- Host Agents (RPM/DEB/MSI)
                          +-- Base: host metrics
                          +-- Extensions: workload collection
